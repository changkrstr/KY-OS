### 2020年 第4题（选择题）

下列与进程调度有关的因素中在设计多级反馈队列调度算法时需要考虑的是（  ）。

I. 就绪队列的数量　　II. 就绪队列的优先级

III. 各就绪队列的调度算法　　IV. 进程在就绪队列间的迁移条件

A. I，II　　B. III，IV　　C. II，III，IV　　D. I，II，III，IV

**【参考答案】D**

**【解析】** 多级反馈队列调度算法是时间片轮转调度算法与优先级调度算法的综合与发展，需要综合考虑：就绪队列的数量（I）、就绪队列的优先级（II）、各就绪队列的调度算法（III）及进程在队列间的迁移条件（IV）。故选D。

### 2020年 第5题（选择题）

某系统中有A、B两类资源各6个，t时刻资源分配及需求情况如下表所示：

| 进程 | A已分配 | B已分配 | A需求总量 | B需求总量 |
|------|--------|--------|----------|----------|
| P1 | 2 | 3 | 4 | 4 |
| P2 | 2 | 1 | 3 | 1 |
| P3 | 1 | 2 | 3 | 4 |

t时刻安全检测结果是：（  ）

A.存在安全序列P1，P2，P3　　B.存在安全序列P2，P1，P3

C.存在安全序列P2，P3，P1　　D.不存在安全序列

**【参考答案】B**

**【解析】**

```
Need = Max - Allocation:
P1: (4-2, 4-3) = (2, 1)
P2: (3-2, 1-1) = (1, 0)
P3: (3-1, 4-2) = (2, 2)

Available = (6,6) - (2+2+1, 3+1+2) = (1, 0)
```

此时Available=(1,0)，只能满足P2的需求(1,0)。释放P2所占资源后：Available=(1,0)+(2,1)=(3,1)，可以满足P1的需求(2,1)。释放P1后：Available=(3,1)+(2,3)=(5,1)，可以满足P3的需求(2,2)。

安全序列为P2→P1→P3。故选B。

### 2019年 第5题（选择题）

系统采用二级反馈队列调度算法进行进程调度。就绪队列Q1采用时间片轮转调度算法，时间片为10ms；就绪队列Q2采用短进程优先调度算法；系统优先调度Q1队列中的进程，当Q1为空时系统才会调度Q2中的进程；新创建的进程首先进入Q1；Q1中的进程执行一个时间片后，若未结束，则转入Q2。若当前Q1、Q2为空，系统依次创建进程P1、P2后即开始进程调度，P1、P2需要的CPU时间分别为30ms和20ms，则进程P1、P2在系统中的平均等待时间为（  ）。

A. 25ms　　B. 20ms　　C. 15ms　　D. 10ms

**【参考答案】C**

**【解析】** 系统依次创建P1、P2，首先进入Q1队列：

1. P1在Q1执行10ms（未完成）→ 转入Q2
2. P2在Q1执行10ms（未完成）→ 转入Q2
3. Q2采用短进程优先，P2(剩余10ms)先于P1(剩余20ms)执行
4. P2执行10ms完成
5. P1再执行20ms完成

P1等待时间 = 10(Q1中P2等待) + 10(P2在Q2执行) = 20ms
P2等待时间 = 10(Q1中P1执行) = 10ms

平均等待时间 = (20+10)/2 = 15ms。故选C。

### 2019年 第8题（选择题）

下列关于死锁的叙述中，正确的是（  ）。

I.可以通过剥夺进程资源解除死锁

II.死锁的预防方法能确保系统不发生死锁

III.银行家算法可以判断系统是否处于死锁状态

IV.当系统出现死锁时，必然有两个或两个以上的进程处于阻塞态

A.仅II、III　　B.仅I、II、IV　　C.仅I、II、III　　D.仅I、III、IV

**【参考答案】B**

**【解析】** 通过剥夺进程资源可以解除死锁（I正确）。死锁的预防方法通过破坏死锁必要条件来确保系统不发生死锁（II正确）。银行家算法是死锁**避免**算法，用于判断系统是否处于安全状态，而非直接判断是否死锁（III错误）。当系统出现死锁时，相互等待的进程必然处于阻塞态，因此必然有两个或两个以上进程处于阻塞态（IV正确）。故选B。

### 2018年 第2题（选择题）

某系统采用基于优先权的非抢占式进程调度策略，完成一次进程调度和进程切换的系统时间开销为1us。在T时刻就绪队列中有3个进程P1、P2和P3，其在就绪队列中的等待时间、需要的CPU时间和优先权如下表所示。

| 进程 | 等待时间 | 需要的CPU时间 | 优先权 |
|------|---------|-------------|--------|
| P1 | 30us | 12us | 10 |
| P2 | 15us | 24us | 30 |
| P3 | 18us | 36us | 20 |

若优先权值大的进程优先获得CPU，从T时刻起系统开始进程调度，则系统的平均周转时间为（   ）。

A.54us　　B.73us　　C.74us　　D.75us

**【参考答案】D**

**【解析】** 优先权值大的进程优先获得CPU，执行顺序为P2(优先权30)→P3(优先权20)→P1(优先权10)。

调度开销1us计入：

- P2的周转时间 = 1(切换) + 15(等待) + 24(CPU) = 40us
- P3的周转时间 = 18(等待) + 1(切换) + 24(P2占用) + 1(切换) + 36(CPU) = 80us
- P1的周转时间 = 30(等待) + 1(切换) + 24(P2占用) + 1(切换) + 36(P3占用) + 1(切换) + 12(CPU) = 105us

平均周转时间 = (40 + 80 + 105)/3 = 75us。故选D。

### 2018年 第4题（选择题）

假设系统中有4个同类资源，进程P1、P2和P3需要的资源数分别为4、3和1，P1、P2和P3已申请到的资源数分别为2、1和0，则执行安全性检测算法的结果（   ）。

A.不存在安全序列，系统处于不安全状态

B.存在多个安全序列，系统处于安全状态

C.存在唯一的安全序列P3、P1、P2，系统处于安全状态

D.存在唯一的安全序列P3、P2、P1，系统处于安全状态

**【参考答案】A**

**【解析】** 系统总共4个资源，已分配2+1+0=3个，剩余1个。P3还需1个，满足P3→P3执行完释放1个→系统共剩余2个。P1还需2个，P2还需2个，都无法满足。因此不存在安全序列，系统处于不安全状态。故选A。

### 2017年 第1题（选择题）

假设4个作业到达系统的时刻和运行时间如下表所示。

| 作业 | 到达时刻t | 运行时间 |
|------|----------|---------|
| J1 | 0 | 3 |
| J2 | 1 | 3 |
| J3 | 1 | 2 |
| J4 | 3 | 1 |

系统在t=2时开始作业调度。若分别采用先来先服务和短作业优先调度算法，则选中的作业分别是（   ）。

A.J2、J3　　B.J1、J4　　C.J2、J4　　D.J1、J3

**【参考答案】D**

**【解析】** t=2时，已到达的作业有J1(运行时间3)、J2(运行时间3)、J3(运行时间2)。

- FCFS：按到达顺序，J2先于J3到达，但J1到达最早(t=0)。不过t=2才开始调度，此时J1在t=0就到达，J1应已在等待。若按FCFS选J1。
- 短作业优先：J3运行时间最短(2)，选J3。

故FCFS选J1，SJF选J3，答案为D（J1、J3）。

### 2014年 第1题（选择题）

下列调整中，不可能导致饥饿现象的是（   ）。

A.时间片轮转　　B.静态优先级调度

C.非抢占式作业优先　　D.抢占式短进程优先

**【参考答案】A**

**【解析】** 时间片轮转方法能在一个周期内使每个进程都得到一个时间片的CPU使用时间，不会产生饥饿现象。静态优先级调度中低优先级进程可能一直得不到调度（B可能饥饿）。非抢占式作业优先中长作业可能一直等待短作业（C可能饥饿）。抢占式短进程优先中长进程可能一直得不到CPU（D可能饥饿）。故选A。

### 2014年 第2题（选择题）

某系统有 n 台互斥使用的同类设备，3个并发进程需要3、4、5台设备，可确保系统发生死锁的设备数n最小为（   ）。

A.9　　B.10　　C.11　　D.12

**【参考答案】B**

**【解析】** 要确保系统发生死锁，考虑每个进程都只差一个资源就能运行完成的情况：每个进程各获得（需要数-1）台设备，即2+3+4=9台。此时再多1个资源就可以打破死锁的条件。因此n最小为9+1=10。即n=10时，10个设备全部分配后各进程分别有2,3,4台，各差1台，系统恰好处于死锁的边缘状态（不安全但不一定死锁），但实际上此时任意一个进程再多获得1台即可打破，所以n=10是确保死锁可能发生的最小值。故选B。

### 2012年 第5题（选择题）

假设5个进程P0、P1、P2、P3、P4共享三类资源R1、R2、R3，这些资源总数分别为18、6、22。T0时刻的资源分配情况如下表所示。

| 进程 | R1已分配 | R2已分配 | R3已分配 | R1最大需求 | R2最大需求 | R3最大需求 |
|------|---------|---------|---------|----------|----------|----------|
| P0 | 3 | 2 | 3 | 5 | 5 | 10 |
| P1 | 4 | 0 | 3 | 5 | 3 | 6 |
| P2 | 4 | 0 | 5 | 4 | 0 | 11 |
| P3 | 2 | 0 | 4 | 4 | 2 | 5 |
| P4 | 3 | 1 | 4 | 4 | 2 | 4 |

此时存在的一个安全序列是（   ）。

A.P0，P2，P4，P1，P3　　B.P1，P0，P3，P4，P2

C.P2，P1，P0，P3，P4　　D.P3，P4，P2，P1，P0

**【参考答案】D**

**【解析】** 系统剩余资源Available = (18-16, 6-3, 22-19) = (2, 3, 3)。

P1需(1,3,3) ≤ (2,3,3)，P1完成后释放→Available=(6,3,6)
P4需(1,1,0) ≤ (6,3,6)，P4完成后释放→Available=(9,4,10)
P2需(0,0,6) ≤ (9,4,10)，P2完成后释放→Available=(13,4,15)
P0需(2,3,7) ≤ (13,4,15)，P0完成后释放→Available=(16,6,18)
P3需(2,2,1) ≤ (16,6,18)，P3完成。

安全序列为P3, P4, P2, P1, P0（也可有其他序列）。注意选项D为P3,P4,P2,P1,P0，但与上述验证的P1先执行不同。实际上根据D选项验证：先执行P3，需(2,2,1)≤(2,3,3)✓。故选D。

### 2011年 第1题（选择题）

下列选项中，满足短任务优先且不会发生饥饿现象的调度算法是（   ）。

A.先来先服务　　B.高响应比优先　　C.时间片轮转　　D.非抢占式短任务优先

**【参考答案】B**

**【解析】** 响应比 = (作业执行时间+作业等待时间)/作业执行时间。高响应比算法在等待时间相同情况下，作业执行时间越少响应比越高，优先执行。随着等待时间增加，响应比也会变大，执行机会增大，不会产生饥饿。先来先服务和时间片轮转不符合短任务优先，非抢占式短任务优先会产生饥饿。故选B。

### 2020年 第11题（8分）

现有5个操作A、B、C、D和E，操作C必须在A和B完成后执行，操作E必须在C和D完成后执行，请使用信号量的wait()、signal()操作（P、V操作）描述上述操作之间的同步关系，并说明所用信号量及其初值。

**【参考答案】**

本题要求实现操作的先后顺序，属于同步问题。

分别设置A、B、C、D、E这5个操作对应的同步信号量为a、b、c、d、e，初始值均为0。

```
Semaphore a=0, b=0, c=0, d=0, e=0;

// 操作A完成后 signal(a)
// 操作B完成后 signal(b)
// 操作C:
//   wait(a);   // 等待A完成
//   wait(b);   // 等待B完成
//   执行操作C;
//   signal(c); // C完成，通知E
// 操作D完成后 signal(d)
// 操作E:
//   wait(c);   // 等待C完成
//   wait(d);   // 等待D完成
//   执行操作E;
//   signal(e); // E完成
```

**【解析】** 这是典型的前趋关系同步问题。信号量a、b用于表示A、B是否完成，c表示C是否完成，d表示D是否完成。每个操作完成后执行signal操作通知后续操作，后续操作执行前执行wait操作等待前置操作完成。

### 2019年 第11题（8分）

有n（n>=3）位哲学家围坐在一张圆桌边，每位哲学家交替地就餐和思考。在圆桌中心有m（m>=1）个碗，每两位哲学家之间有1根筷子。每位哲学家必须取到一个碗和两侧的筷子之后，才能就餐，进餐完毕，将碗和筷子放回原位，并继续思考。为使尽可能多的哲学家同时就餐，且防止出现死锁现象，请使用信号量的P、V操作描述上述过程中的互斥与同步，并说明所用信号量及初值的含义。

**【参考答案】**

```
// 信号量
semaphore bowl;                    // 用于协调哲学家对碗的使用
semaphore chopsticks[n];           // 用于协调哲学家对筷子的使用

for(int i=0; i<n; i++)
    chopsticks[i].value = 1;       // 每两哲学家之间有1根筷子
bowl.value = m;                    // 碗的数量，初始值为m

pi()
{
    while(True) {                  // 哲学家i的程序
        思考;
        P(bowl);                   // 取碗
        P(chopsticks[i]);          // 取左边筷子
        P(chopsticks[(i+1) % n]);  // 取右边筷子
        就餐;
        V(chopsticks[(i+1) % n]);  // 放回右边筷子
        V(chopsticks[i]);          // 放回左边筷子
        V(bowl);                   // 放回碗
    }
}
```

**【解析】** bowl信号量控制同时就餐的哲学家数量不超过m个。chopsticks[i]信号量控制筷子的互斥使用。每位哲学家必须先取碗（P(bowl)），再取左右筷子，全部获得后才能就餐。就餐完毕后按相反顺序释放资源。当m>=n时，所有哲学家可同时就餐；当m<n时，最多m个哲学家同时就餐。

### 2016年 第12题（10分）

某进程调度程序采用基于优先数（priority）的调度策略，即选择优先数最小的进程运行，进程创建时由用户指定一个nice作为静态优先数。为了动态调整优先数，引入运行时间cpuTime和等待时间waitTime，初值均为0。进程处于执行态时，cpuTime定时加1，且waitTime置0；进程处于就绪态时，cpuTime置0，waitTime定时加1。

请回答下列问题：

（1）若调度程序只将nice的值作为进程的优先数，即priority=nice，则可能会出现饥饿现象，为什么？

（2）使用nice、cpuTime和waitTime设计一种动态优先数计算方法，以避免产生饥饿现象，并说明waitTime的作用。

**【参考答案】**

（1）在基于优先数的调度策略中，如果priority=nice，则每个进程的优先数始终不变（静态）。系统中总有优先数小的进程到来，使到达时间早且优先数大的进程长时间得不到调度，始终处于等待态，产生饥饿现象。

（2）动态优先数计算公式为：

**priority = nice + n1 × cpuTime - n2 × waitTime**

其中n1和n2均大于0，分别用于调整cpuTime和waitTime在进程优先数中所占的比例。

**waitTime的作用**：使长时间等待的进程的waitTime不断增大，从而使priority减小（因为减去n2×waitTime），优先数变小意味着优先级变高，最终获得调度机会，从而避免产生饥饿现象。

### 2014年 第12题（8分）

系统中有多个生产者进程和消费者进程，共享一个可以存1000个产品的缓冲区（初始为空），当缓冲区为未满时，生产者进程可以放入一件其生产的产品，否则等待；当缓冲区非空时，消费者进程可以取走一件产品，否则等待。要求一个消费者进程从缓冲区连续取出10件产品后，其他消费者进程才可以取产品，请用信号量P、V（wait、signal）操作实现进程间的互斥和同步。要求写出完整的过程，并指出所用信号量的含义和初值。

**【参考答案】**

```c
buffer array[1000];                 // 存放产品的缓冲区
buffer nextp;                        // 临时存放生产者产品
buffer nextc[10];                    // 临时存放消费者取出的产品
semaphore empty = 1000;              // 空缓冲区数目
semaphore full = 0;                  // 满缓冲区数目
semaphore mutex1 = 1;                // 生产者之间的互斥
semaphore mutex2 = 1;                // 消费者之间的互斥
int in = 0;                          // 生产者存位置
int out = 0;                         // 消费者取位置

Producer()                           // 生产者进程
{
    Produce an item put in nextp;     // 生产产品
    P(empty);                         // 申请空缓冲区
    P(mutex1);                        // 申请使用缓冲区
    array[in] = nextp;                // 存入缓冲区
    in = (in + 1) % 1000;            // 指针后移
    V(mutex1);                        // 释放互斥
    V(full);                          // 增加满缓冲区数
}

Consumer()                           // 消费者进程
{
    P(mutex2);                        // 申请使用缓冲区（一次取10件）
    for(int i = 0; i < 10; i++) {
        P(full);                      // 申请满缓冲区
        nextc[i] = array[out];        // 取出产品
        out = (out + 1) % 1000;      // 指针后移
        V(empty);                     // 释放空缓冲区
    }
    V(mutex2);                        // 释放互斥
    Consume the items in nextc;       // 消费10件产品
}
```

**【解析】** empty和full是同步信号量，分别表示空缓冲区和满缓冲区数量。mutex1和mutex2是互斥信号量，分别保证生产者之间和消费者之间的互斥。消费者通过持有mutex2保证一次连续取出10件产品，其他消费者需等待。这是经典的生产者-消费者问题的变体。

### 2013年 第11题（7分）

某博物馆最多可容纳500人同时参观，有一个出入口，该出入口一次仅允许1人通过。参观者的活动描述如下：

```
Cobegin
参观者进程 i：
{
```

忽略其它进程的影响和系统开销。初始时进程驻留集为空。目前系统空闲页的页框号依次为32、15、21、41。进程P依次访问的<虚拟页号，访问时刻>为<1,1>、<3,2>、<0,4>、<0,6>、<1,11>、<0,13>、<2,14>。

请回答下列问题：

（1）当虚拟页为<0,4>时，对应的页框号是什么？说明理由。

（2）当虚拟页为<1,11>时，对应的页框号是什么？说明理由。

（3）当虚拟页为<2,14>时，对应的页框号是什么？说明理由。

（4）这种方法是否适合于时间局部性好的程序？说明理由。

**（1）页框号为21。**

理由：起始驻留集为空。时刻1访问第1页，分配页框32；时刻2访问第3页，分配页框15；时刻4访问第0页，分配页框21。此时驻留集为{32, 15, 21}，空闲页框链表为41。

**（2）页框号为32。**

**（3）页框号为41。**

**（4）适合。**

系统可用资源R1, R2 = (3, 2)。P2可以调度执行（Need为(2,2)≤(3,2)），执行完毕后系统资源变为(4,3)。但P1需要(6,3)，P3需要(3,4)，系统无法满足任何一个进程的需求。因此不存在安全序列，系统不安全。

**（2）此刻系统尚未发生死锁。**

虽然系统不安全，但P2当前的申请(1,1)可以被满足（Available(3,2)≥(1,1)），P2可以获得资源并执行完毕。这说明不安全状态不一定立即导致死锁，只是存在死锁的风险。但如果不正确分配资源，未来可能会死锁。

### 北京交通大学 2011年 第9题（选择题）

若所有作业同时到达，在各种调度算法中，平均等待时间最短的是（  ）。

A.先来先服务　　B.短作业优先

C.时间片轮转算法　　D.最高响应比优先

**【参考答案】B**

**【解析】** 当所有作业同时到达时，短作业优先（SJF）的平均等待时间最短，也使得平均周转时间最短。这是SJF算法的理论优势。但SJF可能导致长作业长期得不到执行（饥饿）。故选B。

### 北京交通大学 2011年 第1题（计算和编程题）

有5个作业T1、T2、T3、T4、T5，根据经验，其预期运行时间分别是20、4、10、5、3个时间单位，对应的优先级分别是3、5、1、4、2，其中数值越大，优先级越高。若上述作业按顺序几乎同时到达，忽略进程调度开销，请分别按如下调度算法计算进程的平均周转时间：

（1）先来先服务；（4分）

（2）优先级调度；（3分）

（3）时间片轮转（时间片大小为4个时间单位）。（3分）

**【参考答案】**

**（1）先来先服务（FCFS）：**

| 作业 | 到达时间 | 运行时间 | 完成时间 | 周转时间 |
|------|---------|---------|---------|---------|
| T1 | 0 | 20 | 20 | 20 |
| T2 | 0 | 4 | 24 | 24 |
| T3 | 0 | 10 | 34 | 34 |
| T4 | 0 | 5 | 39 | 39 |
| T5 | 0 | 3 | 42 | 42 |

平均周转时间 = (20+24+34+39+42)/5 = 31.8

**（2）优先级调度（数值越大优先级越高）：**

执行顺序：T2(5)→T4(4)→T1(3)→T5(2)→T3(1)

| 作业 | 完成时间 | 周转时间 |
|------|---------|---------|
| T2 | 4 | 4 |
| T4 | 9 | 9 |
| T1 | 29 | 29 |
| T5 | 32 | 32 |
| T3 | 42 | 42 |

平均周转时间 = (4+9+29+32+42)/5 = 23.2

**（3）时间片轮转（q=4）：**

平均周转时间 = (42+8+34+24+19)/5 = 25.4
