### 【2020年 第6题】影响有效访存时间

下列因素影响请求分页系统有效（平均）访存时间的是（  ）。
I. 缺页率
II. 磁盘读写时间
III. 内存访问时间
IV. 执行缺页处理程序的CPU时间

A. II、III
B. I、IV
C. I、III、IV
D. I、II、III、IV

**【参考答案】D**

**【解析】**
- I. 缺页率影响缺页中断发生的频率，直接影响有效访存时间；
- II. 磁盘读写时间影响页面调入和访问物理地址的访问时间；
- III. 内存访问时间是有效访存时间的基础组成部分；
- IV. CPU时间影响缺页中断的处理时间。

四项均影响有效访存时间，故选D。

### 【2019年 第7题】LRU页置换次数

某系统采用LRU页置换算法和局部置换策略，若系统为进程P预分配了4个页框，进程P访问页号的序列为0，1，2，7，0，5，3，5，0，2，7，6，则进程访问上述页的过程中，产生页置换的总次数是（  ）。

A. 3
B. 4
C. 5
D. 6

**【参考答案】C**

**【解析】**
通过LRU页置换算法，逐步分析页面访问过程：

| 访问页 | 内存状态（LRU在右） | 是否缺页 | 是否置换 |
|:---:|:---|:---:|:---:|
| 0 | {0} | 缺页 | 否（有空闲页框） |
| 1 | {0, 1} | 缺页 | 否 |
| 2 | {0, 1, 2} | 缺页 | 否 |
| 7 | {0, 1, 2, 7} | 缺页 | 否 |
| 0 | {1, 2, 7, 0} | 命中 | - |
| 5 | {2, 7, 0, 5} | 缺页 | **置换1** |
| 3 | {7, 0, 5, 3} | 缺页 | **置换2** |
| 5 | {7, 0, 3, 5} | 命中 | - |
| 0 | {7, 3, 5, 0} | 命中 | - |
| 2 | {3, 5, 0, 2} | 缺页 | **置换3** |
| 7 | {5, 0, 2, 7} | 缺页 | **置换4** |
| 6 | {0, 2, 7, 6} | 缺页 | **置换5** |

页面置换的总次数为**5次**，故选C。

### 【2018年 第11题】虚拟地址PDBR/CLOCK字段

请根据下图给出的虚拟存储管理方式，回答下列问题。

（1）某虚拟地址对应的页目录号为6，在相应的页表中对应的页号为6，页内偏移量为8，该虚拟地址的十六进制表示是什么？

（2）寄存器PDBR用于保存当前进程的页目录起始地址，该地址是物理地址还是虚拟地址？进程切换时，PDBR的内容是否会变化？说明理由。同一进程的线程切换时，PDBR的内容是否会变化？说明理由。

（3）为了支持改进型CLOCK置换算法，需要在页表项中设置哪些字段。

**【参考答案与解析】**

**（1）**
页目录号为6（高10位），页号为6（中间10位），页内偏移量为8（低12位），用二进制表示为：
00 0000 0110 | 00 0000 0110 | 0000 0000 1000
用十六进制表示为 **01806008H**。

**（2）**
寄存器PDBR用于保存当前进程的页目录起始地址，该地址是**物理地址**。

**进程切换时，PDBR的内容会变化。** 因为在进程切换时，每一个进程对应的地址空间不同，它们的页目录在内存中的存放位置也是不同的。

**同一进程的线程切换时，PDBR的内容不会变化。** 因为同一个进程中的线程共享同一地址空间，它们对应的页目录是一样的。

**（3）**
为了支持改进型CLOCK置换算法，需要在页表项中设置：
- **访问字段（引用位/使用位A）**：记录该页最近是否被访问过；
- **修改位（M位）**：记录该页在调入内存后是否被修改过。

### 【2015年 第5题】改进型CLOCK淘汰次序

某系统采用改进型CLOCK置换算法，页表项中字段A为访问位，M为修改位。A=0表示页最近没有被访问，A=1表示页最近被访问过；M=0表示页没有被修改过，M=1表示页被修改过。按（A,M）所有可能的取值，将页分为四类：（0,0）、（1,0）、（0,1）和（1,1），则该算法淘汰页的次序为（  ）。

A. (0,0)，(0,1)，(1,0)，(1,1)
B. (0,0)，(1,0)，(0,1)，(1,1)
C. (0,0)，(0,1)，(1,1)，(1,0)
D. (0,0)，(1,1)，(0,1)，(1,0)

**【参考答案】A**

**【解析】**
改进型CLOCK置换算法将页面分为四种类型，淘汰优先级如下：
1. **(A=0, M=0)**：该页最近既未被访问，也未被修改，是最佳淘汰页（最先淘汰）；
2. **(A=0, M=1)**：该页最近未被访问，但已被修改，并不是很好的淘汰页（第二轮淘汰）；
3. **(A=1, M=0)**：该页最近已被访问，但未被修改，有可能再被访问（第三轮淘汰）；
4. **(A=1, M=1)**：该页最近已被访问且被修改，可能再被访问（最后淘汰）。

故淘汰次序为 (0,0)→(0,1)→(1,0)→(1,1)，选A。

### 【2015年 第6题】TSL伪代码正确性

使用TSL（Test and Set Lock）指令实现进程互斥的伪代码如下所示：
```
do{
    ...
    while(TSL(&lock));
    critical section;
    lock = FALSE;
    ...
} while(TRUE);
```
下列与该实现机制相关的叙述中，正确的是（  ）。

A. 退出临界区的进程负责唤醒阻塞态进程
B. 等待进入临界区的进程不会主动放弃CPU
C. 上述伪代码满足"让权等待"的同步准则
D. while(TSL(&lock))语句应在关中断状态下执行

**【参考答案】B**

**【解析】**
- A错误：当进程退出临界区时，将lock置为FALSE，此时就绪态进程可以竞争进入临界区，但退出临界区的进程并不负责"唤醒"阻塞态进程。
- B正确：若等待进入临界区的进程一直停留在while循环中（忙等待），则不会主动放弃CPU。
- C错误：进程一直停留在while循环中不会释放处理机，上述伪代码不满足"让权等待"的同步准则。
- D错误：若while(TSL(&lock))语句在关中断状态下执行，当TSL(&lock)一直为TRUE时不再开中断，系统可能会因此终止。

某进程访问页面的序列如下所示（窗口大小为6）。若工作集的窗口大小为6，则在t时刻的工作集为（  ）。

A. {6，0，3，2}

### 【2015年 第9题】页面置换分配策略

在请求分页系统中，页面分配策略与页面置换策略不能组合使用的是（  ）。

A. 可变分配，全局置换
B. 可变分配，局部置换
C. 固定分配，全局置换
D. 固定分配，局部置换

**【参考答案】C**

**【解析】**
固定分配是指为每个进程分配一定数目的物理块，在整个运行期间都不再改变。采用该策略时，如果进程在运行中发现缺页，则只能从该进程在内存的n个页面中选出一个页换出，然后再调入一页，才能保证分配给该进程的内存空间不变。因此**不能**有"固定分配+全局置换"的组合。因为全局置换意味着可以从其他进程的页框中取页，这与"固定分配"矛盾。

### 【2015年 第10题】位示图磁盘空间

文件系统用位图法表示磁盘空间的分配情况，位图存于磁盘的32~127号块中，每个盘块占1024个字节，盘块和块内字节均从0开始编号。假设要释放的盘块号为409612，则位图中要修改的位所在的盘块号和块内字节序号分别是（  ）。

A. 81、1
B. 81、2
C. 82、1
D. 82、2

**【参考答案】C**

**【解析】**
位图中每个位对应一个盘块号。盘块号409612对应的位在位图中的第 409612 / 8 = 51201 个字节中（从0开始），该字节内的位序号为 409612 mod 8 = 4。

位图存于磁盘的32~127号块中，每个盘块1024字节，所以位图共有 (127-32+1) × 1024 = 96 × 1024 = 98304 个字节。

51201 / 1024 = 50，余 51201 - 50 × 1024 = 1。

从32号块开始计算，盘块号 = 32 + 50 = **82**，块内字节序号为 **1**。

故选C：82、1。

### 【2014年 第5题】位图法簇分配

现有容量为10GB的磁盘分区，磁盘空间以簇（cluster）为单位进行分配，簇的大小为4KB，若采用位图法管理该分区的空闲空间，即用1位（bit）标识一个簇是否被分配，则存放该位图所需簇的个数为（  ）。

A. 80
B. 320
C. 80K
D. 320K

**【参考答案】A**

**【解析】**
设磁盘容量为A，则 A = 10G = 10 × 1024 × 1024 × 1024 字节。
簇大小 B = 4KB = 4 × 1024 字节。
需要标识的簇数 C = A / B = 10 × 1024 × 1024 × 1024 / (4 × 1024) = 10 × 1024 × 1024 / 4 = 2.5M 个簇。

每个盘块可存放的位数 = 1024 × 8 = 8192 位。
需要的位图盘块数 = 2.5M / 8192 ≈ **80个**。

故选A。

### 【2014年 第8题】Belady异常

在页式存储管理系统中，采用某些页面置换算法，会出现Belady异常现象，即进程的缺页次数会随着分配给该进程的页框个数的增加而增加。下列算法中，可能出现Belady异常现象的是（  ）。

I. LRU算法
II. FIFO算法
III. OPT算法

A. 仅II
B. 仅I、II
C. 仅I、III
D. 仅II、III

**【参考答案】A**

**【解析】**
Belady异常现象是系统为进程分配的页数增多，但缺页率反而提高的异常现象。在上述算法中，只有**FIFO算法**会发生Belady异常。LRU算法和OPT算法都是栈式算法，不会出现Belady异常。

### 【2013年 第5题】单缓冲/双缓冲时间计算

设系统缓冲区和用户工作区均采用单缓冲，从外设读入1个数据块到系统缓冲区的时间为100，从系统缓冲区读入1个数据块到用户工作区的时间为5，对用户工作区中的1个数据块进行分析的时间为90。进程从外设读入并分析2个数据块的最短时间是（  ）。

A. 200
B. 295
C. 300
D. 390

**【参考答案】C**

**【解析】**
单缓冲情况下，处理过程如下：
- 数据块1从外设→系统缓冲区（100），再从系统缓冲区→用户工作区（5），然后CPU分析（90）。在数据块1分析期间（90时间），数据块2可以从外设→系统缓冲区（100，但与分析并行，只需取较大值）。
- 数据块1总耗时：100 + 5 + 90 = 195
- 数据块2在数据块1分析的同时可以开始读入，但由于单缓冲限制，数据块2从缓冲区→用户工作区须等数据块1分析完。
- 总时间 = 100 + 5 + 90 + 5 + 90 = **300**

### 【2011年 第7题】抖动有效措施

当系统发生抖动（thrashing）时，可采取的有效措施是（  ）。
I. 撤销部分进程
II. 增加磁盘交换区的容量
III. 提高用户进程的优先级

A. 仅I
B. 仅II
C. 仅III
D. 仅I、II

**【参考答案】A**

**【解析】**
抖动现象是指刚刚被换出的页很快又要被访问，频繁的置换页面，以致大部分时间都花在页面置换上。
- I正确：**撤销部分进程**可以减少所要用到的页面数，防止抖动。
- II错误：增加磁盘交换区的容量只是增加了外存空间，不能减少页面置换频率，对抖动无效。
- III错误：提高用户进程的优先级与抖动无关。

### 【2011年 第9题】单缓冲/双缓冲读取文件时间

某文件占10个磁盘块，现要把该文件磁盘块逐个读入主存缓冲区，并送用户区进行分析，假设一个缓冲区与一个磁盘块大小相同，把一个磁盘块读入缓冲区的时间为100us，将缓冲区的数据传送到用户区的时间是50us，CPU对一块数据进行分析的时间为50us。在单缓冲区和双缓冲区结构下，读入并分析完该文件的时间分别是（  ）。

A. 1500us、1000us
B. 1550us、1550us
C. 1550us、1100us
D. 2000us、2000us

**【参考答案】C**

**【解析】**

**单缓冲区：**
- 每个磁盘块：读入缓冲区(100us) + 传送到用户区(50us) = 150us（顺序执行，不能重叠）
- 10个磁盘块共需：150 × 10 = 1500us
- 加上最后一个块的CPU分析时间：50us
- 总时间 = 1500 + 50 = **1550us**

**双缓冲区：**
- 两个缓冲区交替使用，当一个缓冲区向用户区传送数据时，另一个缓冲区可以同时从磁盘读入数据。
- 磁盘可以连续读入10个块：100 × 10 = 1000us
- 最后一个块从缓冲区传到用户区：50us
- CPU分析最后一个块：50us
- 总时间 = 1000 + 50 + 50 = **1100us**

### 【2015年 第13题】二级页表页/页框大小/虚拟空间

某计算机系统按字节编址，采用二级页表的分页存储管理方式，虚拟地址格式如下所示：

| 页目录号（10位） | 页表索引（10位） | 页内偏移量（12位） |
|:---:|:---:|:---:|

请回答下列问题：

（1）页和页框的大小各为多少字节？进程的虚拟地址空间大小为多少页？

（2）假定页目录项和页表项均占4个字节，则进程的页目录和页表共占多少页？要求写出计算过程。

（3）若某指令周期内访问的虚拟地址为0100 0000H和0111 2048H，则进行地址转换时共访问多少个二级页表？要求说明理由。

**【参考答案与解析】**

**（1）**
- 页内偏移量为12位，所以页和页框的大小均为 2^12B = **4KB**。
- 地址空间共32位，每页占12位，故进程的虚拟地址空间大小为 2^32 / 2^12 = **2^20 页**（即1M页）。

**（2）**
- 页目录所占页数 = (2^10 × 4) / 2^12 = 4KB / 4KB = **1页**
- 页表所占页数 = (2^20 × 4) / 2^12 = 4MB / 4KB = **1024页**
- 页目录和页表共占 1 + 1024 = **1025页**

**（3）**
共访问**1个**二级页表。
- 虚拟地址 0100 0000H 的高10位（页目录号）为 0000 0100 00 = 4
- 虚拟地址 0111 2048H 的高10位（页目录号）为 0000 0100 00 = 4
- 两个虚拟地址的页目录号相同，故访问的是同一个二级页表。

### 【2013年 第12题】页表项物理地址/页框号

某计算机主机按字节编址，逻辑地址和物理地址都是32位，页表项大小为4字节。请回答下列问题。

（1）若使用一级页表的分页存储管理方式，逻辑地址结构如下：

| 页号（20位） | 页内偏移量（12位） |
|:---:|:---:|

则页的大小是多少字节？页表最大占用多少字节？

（2）若使用二级页表的分页存储管理方式，逻辑地址结构如下：

| 页目录号（10位） | 页表索引（10位） | 页内偏移量（12位） |
|:---:|:---:|:---:|

设逻辑地址为LA，请分别给出其对应的页目录号和页表索引的表达式。

（3）采用（1）中的分页存储管理方式，一个代码段起始逻辑地址为0000 8000H，其长度为8KB，被装载到从物理地址0090 0000H开始的连续主存空间中。页表从主存0020 0000H开始的物理地址处连续存放，请计算出该代码段对应的2个页表项的物理地址，这2个页表项中的页框号以及代码页面2的起始物理地址。

**【参考答案与解析】**

**（1）**
- 页内偏移量是12位，故页的大小为 2^12B = **4KB**。
- 页表项数 = 2^20，故页表最大占用 2^20 × 4B = **4MB**。

**（2）**
- 页目录号 = {[(unsigned int)(LA)] >> 22} & 0x3FF
- 页表索引 = {[(unsigned int)(LA)] >> 12} & 0x3FF

**（3）**
代码段的起始逻辑地址为 0000 8000H：
- 页号 = 0 0008H，即位于第8页中，对应页表中的第8个页表项
- 页表项1的物理地址 = 起始物理地址 + 页号 × 页表项大小 = 0020 0000H + 8 × 4 = **0020 0020H**
- 代码段长度为8KB，页大小为4KB，需用2个页存放
- 页表项2的物理地址 = 0020 0020H + 1 × 4 = **0020 0024H**
- 代码段装载到物理地址 0090 0000H 开始的连续空间中，页框号1 = 00900H
- 页框号2 = 00900H + 1 = **00901H**
- 代码页面2的起始物理地址 = 00901H × 4KB = **0090 1000H**
